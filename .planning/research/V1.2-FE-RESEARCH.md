# Phase Research: V1.2 Update the FE

**Researched:** May 2026
**Domain:** Frontend Development, API Integration, UI/UX, Testing
**Confidence:** HIGH

## Summary

The current frontend (`fe/src`) is a high-fidelity React 19 application with a polished "Glassmorphism" aesthetic. However, it is largely decoupled from the backend, relying heavily on local state and mock data for core features such as Authentication, AI Generation, Idea Management, and Analytics. The primary objective of Milestone v1.2 is to bridge this gap by connecting the UI to the existing .NET backend and enhancing the user experience with "Pro Max" animations and micro-interactions.

**Primary recommendation:** Implement a robust API layer using Axios (replacing the current minimal `fetch` wrapper) to handle JWT authentication and workspace-scoped requests, while simultaneously integrating Framer Motion for premium-feel UI transitions.

## Architectural Responsibility Map

## Architecture Patterns

### Recommended Project Structure
```
fe/src/
├── api/             # Axios instances and API definitions
├── components/      # Pure UI and Shared components
├── context/         # Global state (Auth, Workspace, Billing)
├── hooks/           # Custom hooks (usePosts, useAnalytics)
├── pages/           # Page-level components
└── services/        # Business logic wrappers
```

### Pattern 1: Optimistic Updates
**What:** Immediately update the UI for Idea movement (drag-and-drop) before the API call finishes.
**When to use:** Ideas Board, Calendar rescheduling.

### Pattern 2: Higher-Order Auth Wrapper
**What:** Protecting routes using a component that checks for valid JWT.
**When to use:** All `/app/*` routes.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Date Formatting | Custom string parsing | `date-fns` | Complex timezone and locale issues. |
| Drag and Drop | Native HTML5 DnD | `@dnd-kit` | Handles accessibility and touch better (already in project). |
| Form Validation | Manual state checks | `Zod` + `React Hook Form` | Edge cases in input types and validation logic. |
| Infinite Scroll | Manual scroll listeners | `react-intersection-observer` | Performance and reliability. |

## Common Pitfalls

### Pitfall 1: Z-Index Wars in Portals
**What goes wrong:** Modals and Dropdowns overlapping incorrectly or being cut off.
**Why it happens:** Inconsistent use of `createPortal`.
**How to avoid:** Use a dedicated `DropdownPortal` component (exists) and standard stack layers.

### Pitfall 2: Token Refresh Race Conditions
**What goes wrong:** Multiple failed requests trigger multiple refresh token calls simultaneously.
**Why it happens:** Standard Axios interceptors without request queuing.
**How to avoid:** Implement a "isRefreshing" flag and request queue in Axios interceptor.

## Pro Max UI/UX Enhancement Strategy

1.  **Page Transitions:** Use Framer Motion `<AnimatePresence>` for smooth "Slide & Fade" transitions between sidebar links.
2.  **Staggered Board Loading:** Instead of a static board, animate cards entering columns with a slight delay (stagger).
3.  **Skeleton Loaders:** Implement custom glass-themed skeleton loaders for Dashboard stats and Table rows.
4.  **Micro-interactions:** Add haptic-style animations to buttons (scale down on click, scale up on hover).
5.  **Interactive Heatmap:** Ensure the Analytics heatmap is interactive, highlighting specific cells on hover with detailed tooltips.
6.  **Progressive AI Reveal:** Animate AI-generated text using a "typewriter" effect or a gradient shimmer while "loading."

## Missing Features (Required by Backend)

1.  **Workspace Switcher:** The backend supports multiple workspaces, but the frontend lacks a UI to switch between them.
2.  **Social Account Health:** UI to display `ok`, `warning`, `needs_reauth` statuses from `/health` endpoint.
3.  **Real AI Connection:** The frontend currently mocks AI generation; must decide if this moves to backend or remains a direct OpenAI call from frontend (not recommended).
4.  **Media Management:** A centralized view to manage files uploaded via `/api/v1/workspaces/{id}/media`.
5.  **Onboarding Progress Persistence:** Move onboarding status from `localStorage` to the User Profile in backend.
