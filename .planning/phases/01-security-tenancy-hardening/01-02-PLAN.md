  <files>src/Syncra.Api/Controllers/StripeWebhookController.cs, tests/Syncra.UnitTests/Api/StripeWebhookControllerTests.cs</files>
  <behavior>
    - New event (evt_123):
        - Create record: Key='stripe_event_evt_123', RequestHash='evt_123', Endpoint='/api/webhooks/stripe', Method='POST', Status='Pending', ExpiresAtUtc=Now+7d.
        - Process -> Success -> Record.Status='Success' -> Return 200.
    - Duplicate event (evt_123) while Status='Pending':
        - Return 409 Conflict (prevents concurrent processing).
    - Duplicate event (evt_123) while Status='Success':
        - Return 200 OK immediately (no re-processing).
    - Duplicate event (evt_123) while Status='Failed':
        - Allow retry: Update Status='Pending' and re-process.
  </behavior>
  <action>
    Modify `StripeWebhookController.Index` per 1-RESEARCH.md:
    - Inject `AppDbContext`.
    - Extract `eventId = stripeEvent.Id`.
    - Query `db.IdempotencyRecords` for `Key == $"stripe_event_{eventId}"`.
    - If record exists:
        - If `Status == "Pending"`, return `Conflict("Event processing in progress")`.
        - If `Status == "Success"`, return `Ok()`.
        - If `Status == "Failed"`, reset `Status = "Pending"` and continue to processing.
    - If record does not exist, create new `IdempotencyRecord` with all mandatory fields:
        - `Key`: `$"stripe_event_{eventId}"`
        - `RequestHash`: `eventId`
        - `Endpoint`: `"/api/webhooks/stripe"`
        - `Method`: `"POST"`
        - `Status`: `"Pending"`
        - `ExpiresAtUtc`: `DateTime.UtcNow.AddDays(7)`
        - `WorkspaceId`: Extract from Stripe metadata/reference.
    - `await db.SaveChangesAsync()`.
    - Wrap event switch in try-catch:
        - On Success: `record.Status = "Success"`.
        - On Failure: `record.Status = "Failed"`.
    - Final `await db.SaveChangesAsync()`.
  </action>
