<objective>
Implement the Phase 7 Billing UI inside **Settings → Billing**:
- show current plan + status + next billing/trial end
- in-app pricing cards with Monthly/Yearly toggle
- Checkout CTA (owner-only) using v2 checkout endpoint
- Manage Billing CTA (owner-only) using Stripe customer portal
- return banners using `?billing=success|cancel|portal_return` and auto-refresh subscription state

This plan consumes `BillingContext` from Plan 07-02.
</objective>

<threat_model>
## Threats

### T1: UI exposes privileged CTAs to non-owner
- **Severity:** HIGH
- **Mitigation:** Hide checkout/portal CTAs unless `isOwner === true`.

### T2: Redirect UX hides failures
- **Severity:** MEDIUM
- **Mitigation:** show explicit error banners when API calls fail (no silent failures).

### T3: Phishing-like redirect links
- **Severity:** LOW
- **Mitigation:** redirects only use URLs returned from backend; UI must not accept arbitrary redirect URLs from query params.
</threat_model>

<action>
Create `fe/src/components/billing/BillingSection.tsx` implementing:

1) **Data load**
- On mount, call `loadCurrentSubscription()` from `useBilling()`.

2) **Return banner**
- Use `useSearchParams()` (react-router) to read query param `billing`.
- If `billing` is `success`, `cancel`, or `portal_return`, render a dismissible inline banner at the top of the Billing section.
- If `billing` is `success` or `portal_return`, call `loadCurrentSubscription()` again to auto-refresh.
- After banner is shown, remove the `billing` query param via router replace.

3) **Subscription summary**
- Display Plan name, Status, and either Trial end or Next billing date.
- Use API fields:
  - show `trialEndsAtUtc` when present
  - else show `endsAtUtc` when present

4) **Owner gating (temporary FE-only)**
- Determine `isOwner` via localStorage:
  - `const isOwner = localStorage.getItem('syncra_is_owner') !== 'false'`
  - (So default is owner; setting the key to `false` simulates a non-owner)
- If `isOwner === false`, hide checkout + portal CTAs and show helper text: `Only the workspace owner can manage billing actions.`

Also render errors from BillingContext visibly.
</action>
<acceptance_criteria>
- `BillingSection.tsx` contains `billing=success` OR checks `searchParams.get('billing')`
- `BillingSection.tsx` contains string `Only the workspace owner can manage billing actions.`
- `BillingSection.tsx` contains `syncra_is_owner`
- `BillingSection.tsx` calls `loadCurrentSubscription` in a `useEffect`
- `BillingSection.module.css` defines a banner style class (e.g. `.banner`)
</acceptance_criteria>
