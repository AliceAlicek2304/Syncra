<objective>
Create the frontend “Billing data layer” used by Settings → Billing:
- a tiny API wrapper (auth + tenant header)
- TypeScript DTO types for subscription + checkout/portal session
- a unified `BillingContext` that exposes: load subscription, start checkout, open portal

This plan intentionally focuses on **data + state**, not UI polish.
</objective>

<threat_model>
## Threats

### T1: Token leakage in frontend logs
- **Severity:** MEDIUM
- **Mitigations:**
  - Never log raw Authorization tokens
  - Keep token retrieval isolated in API helper

### T2: Cross-tenant request mistakes
- **Severity:** HIGH
- **Mitigations:**
  - Always send `X-Workspace-Id` header
  - Centralize header logic in the API helper

### T3: Silent failures (bad UX)
- **Severity:** MEDIUM
- **Mitigations:**
  - API helper throws structured errors
  - BillingContext exposes `error` string; UI must render it
</threat_model>

<action>
Create `fe/src/context/BillingContext.tsx`:

- Provide `BillingProvider` + `useBilling()` hook
- Context state:
  - `subscription: CurrentSubscriptionDto | null`
  - `loading: boolean`
  - `error: string | null`
  - `redirecting: boolean`

- Context helpers:
  - `getWorkspaceId(): string | null` reads `localStorage.getItem('syncra_workspace_id')`

- Context actions:
  1) `loadCurrentSubscription()`
     - Requires workspaceId; if missing set error
     - Calls `GET /api/v1/workspaces/{workspaceId}/subscription` via `apiFetch` with `workspaceId` header
  2) `startCheckout(planCode: string, interval: 'month'|'year')`
     - Calls `POST /api/v2/workspaces/{workspaceId}/subscription/create-checkout-session`
     - Compute absolute return URLs using Vite base + origin:
       ```ts
       const origin = window.location.origin
       const base = import.meta.env.BASE_URL.replace(/\/$/, '') // "/Syncra" in this repo
       const successUrl = new URL(`${base}/app/settings?billing=success`, origin).toString()
       const cancelUrl  = new URL(`${base}/app/settings?billing=cancel`, origin).toString()
       ```
     - Send `successUrl` + `cancelUrl` in the POST body.
     - On success, set `redirecting=true` then `window.location.assign(checkoutUrl)`.

  3) `openPortal()`
     - Calls `POST /api/v1/workspaces/{workspaceId}/subscription/create-portal-session` (or v2 if Plan 01 adds it)
     - Compute return URL:
       ```ts
       const origin = window.location.origin
       const base = import.meta.env.BASE_URL.replace(/\/$/, '')
       const returnUrl = new URL(`${base}/app/settings?billing=portal_return`, origin).toString()
       ```
     - Send `returnUrl` in the POST body.
     - On success, set `redirecting=true` then `window.location.assign(portalUrl)`.

- Expose these actions from the hook.

Leave the UI rendering to Plan 07-03.
</action>
