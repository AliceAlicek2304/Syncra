<objective>
Harden backend billing endpoints for Phase 7 UX:
- Enforce tenant header usage (`X-Workspace-Id`) and prevent workspaceId mismatch.
- Enforce **workspace-owner-only** access for billing actions (checkout + portal session creation).
- (Optional but recommended) add a v2 portal-session endpoint for API symmetry.

This ensures the frontend Billing UX is secure-by-default and aligns with Phase 7 decision D-02.
</objective>

<threat_model>
## Threats

### T1: Unauthorized billing actions (non-owner or wrong workspace)
- **Scenario:** authenticated member (or attacker with token) starts checkout/portal for a workspace they should not control.
- **Severity:** HIGH
- **Mitigations:**
  - Require `X-Workspace-Id` header and enforce it matches route `{workspaceId}`.
  - Validate workspace ownership in the application handlers before creating sessions.

### T2: Open redirect / malicious return URL
- **Scenario:** caller passes attacker-controlled `SuccessUrl`/`CancelUrl`/`ReturnUrl` to Stripe session creation.
- **Severity:** MEDIUM
- **Mitigations (Phase 7):**
  - Document safe URL construction in phase docs.
  - (Optional follow-up) backend allow-list of hostnames or force server-computed return URLs.

### T3: Information disclosure via GET subscription
- **Scenario:** user fetches current subscription for workspace they do not belong to.
- **Severity:** HIGH
- **Mitigations:**
  - Require `X-Workspace-Id` header and ensure membership is validated by `TenantResolutionMiddleware`.
</threat_model>

In both subscription controllers, enforce that requests include a validated tenant context:

1) In each action, check `HttpContext.Items[TenantResolutionMiddleware.WorkspaceIdKey]` exists and is a `Guid`.
2) If missing → return 400 with message `X-Workspace-Id header is required.`
3) If present but does not equal the route `workspaceId` → return 400 with message `X-Workspace-Id must match route workspaceId.`

Apply to these actions:
- `SubscriptionsController.GetCurrentSubscription`
- `SubscriptionsController.CreateCheckoutSession`
- `SubscriptionsController.CreatePortalSession`
- `SubscriptionsV2Controller.CreateCheckoutSession`
- (If added) `SubscriptionsV2Controller.CreatePortalSession`

Notes:
- This makes membership validation mandatory via `TenantResolutionMiddleware`.
- Keep error payload consistent with existing middleware style: `{ statusCode, message }`.
