## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| External provider -> webhook ingress | Incoming webhook payloads are untrusted until provider parsing/verification succeeds. |
| Provider parser -> application dispatcher | Only normalized, verified events should reach subscription business logic. |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Spoofing | Webhook ingress | mitigate | Provider-owned signature verification stays inside `IPaymentProvider.ParseWebhookAsync`; legacy Stripe route still rejects missing signature headers. |
| T-04-03-02 | Replay | Idempotency records | mitigate | Canonical idempotency key remains `{provider}_event_{eventId}` with existing pending/success/failure semantics. |
| T-04-03-03 | Tampering | Subscription mutation dispatch | mitigate | Dispatcher emits provider-aware commands containing provider and provider subscription id, preventing ambiguous Stripe-only lookups. |
</threat_model>
