# Phase 22 Plan 02: Email Verification Backend API - Wave 2 Summary

## Objective

Create VerifyEmailCommand + handler, update RegisterCommandHandler to send verification email, update OAuthLoginCommandHandler to auto-verify OAuth users, add API endpoints for token verification and resend, and implement ResendVerificationEmailCommand with rate limiting.

## One-Liner

## Artifacts Delivered

### Commands
- **VerifyEmailCommand** - record with Token property, returns AuthResponseDto (auto-login)
- **VerifyEmailCommandHandler** - validates token (not null, not expired, not used), calls user.VerifyEmail(), marks token as used, generates JWT session, returns AuthResponseDto
- **VerifyEmailCommandValidator** - validates Token is not empty and minimum 10 chars
- **ResendVerificationEmailCommand** - record with Email property, returns ResendVerificationEmailResponse
- **ResendVerificationEmailCommandHandler** - rate limited (1 per 60s via IDistributedCache), looks up user by email, revokes old tokens, generates new token, sends verification email
- **ResendVerificationEmailCommandValidator** - validates email format

### API Endpoints
- `POST /api/v1/auth/verify-email` - Accepts token, validates, auto-logs in, returns JWT
- `POST /api/v1/auth/resend-verification-email` - Accepts email, generates new token with rate limiting

### Key Integration Points
- RegisterCommandHandler sends verification email after user creation (email/password path)
- OAuthLoginCommandHandler calls user.VerifyEmail() for OAuth users (auto-verified by Google)
- Both endpoints are [AllowAnonymous]
